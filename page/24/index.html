错误二：

正在检查网络配置要求…
检查完成。此次检查的总体结果为: 失败 <<<<
问题: 安装检测到系统的主 IP 地址是 DHCP 分配的地址。
建议案: Oracle 支持在具有 DHCP 分配的 IP 地址的系统上进行安装。但在安装之前, 必须将 Microsoft LoopBack Adapter 配置为系统的主网络适配器。有关在配置有 DHCP 的系统上安装软件的详细信息, 请参阅 Installation Guide。

解决方案：

1、开始-运行，输入hdwwiz.exe(win7系统)打开硬件添加界面；xp系统中点击控制面板，然后点击添加硬件
2、选择 手动从列表选择的硬件添加
3、选择”网络适配器”
4、左边选择”Microsoft”，右边选择”Microsoft Loopback Adapter”下一步，确定
5、打开网络连接那里的”更改适配器设置”，给新添加的假网卡”Microsoft Loopback Adapter”随便配置一个IP

SQL 连接字符串

//sql server 身份验证 连接字符串 private string ConnectionStrSqlServer = “server=服务器名称;database=数据库名称;uid=登录名称;pwd=登录密码;”;

//windows 身份验证连接字符串 private string ConnectionStrWindows = “server=服务器名称;database=数据库名称;Trusted_Connection=SSPI;”;

// 在 C# 代码中用 SqlClient 的方式访问 SQL Server 2008 数据库

// .NET Framework Data Provider for SQL Server 标准写法
Data Source=. 或者ip地址(需要允许远程连接);Initial Catalog=DataBase;User Id=UserName;Password=pwd;

// .NET Framework Data Provider for SQL Server 另一种标准写法
Server=. 或者ip地址(需要允许远程连接);Database=DataBase;User ID=UserName;Password=pwd;Trusted_Connection=False;

// .NET Framework Data Provider for SQL Server 信任连接写法
Data Source=. 或者ip地址(需要允许远程连接);Initial Catalog=DataBase;Integrated Security=SSPI;

// .NET Framework Data Provider for SQL Server 信任连接另一种写法
Server=. 或者ip地址(需要允许远程连接);Database=DataBase;Trusted_Connection=True;
