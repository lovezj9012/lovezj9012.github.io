Stage模型

Stage模型概述

  • AblityStage:应用组件的“舞台”
  • UIAbility:包含UI界面的应用组件，是系统调度的基本单元
  • WindowStage：组件内窗口的“舞台”
  • Window：用来绘制UI页面的窗口

Stage模型应用配置文件

配置文件

AppScore下的app.json5配置

该配置文件是全局配置

{
  "app": {
    "bundleName": "com.zj.demo", //唯一标识
    "vendor": "example",
    "versionCode": 1000000, //版本
    "versionName": "1.0.0", //版本
    "icon": "$media:app_icon", //应用图标
    "label": "$string:app_name" //应用描述
  }
}

entry目录下的module.json5

{
  "module": {
    "name": "entry", //当前模块名称
    "type": "entry", //当前模块类型，类型可分两大类：Ability和Library；Ability分为entry和feature，Library分为Shared
    "description": "$string:module_desc", //当前模块的描述
    "mainElement": "EntryAbility", //当前模块的入口
    "deviceTypes": [ //支持设备类型
      "phone",
      "tablet"
    ],
    //网络权限配置
    "requestPermissions": [
      {
        "name": "ohos.permission.INTERNET"
      }
    ],
    "deliveryWithInstall": true, //当前模块类型如果是feature，设置为true表示该模块要跟随一起安装
    "installationFree": false,
    "pages": "$profile:main_pages", //包含页面
    "abilities": [ 
      {
        "name": "EntryAbility",
        "srcEntry": "./ets/entryability/EntryAbility.ts",
        "description": "$string:EntryAbility_desc",
        "icon": "$media:icon", //当前应用桌面的图标
        "label": "$string:EntryAbility_label", //当前应用桌面的描述
        "startWindowIcon": "$media:icon",
        "startWindowBackground": "$color:start_window_background",
        "exported": true,
        "skills": [
          {
            "entities": [
              "entity.system.home"
            ],
            "actions": [
              "action.system.home"
            ]
          }
        ]
      }
    ]
  }
}

UIAbility生命周期

页面及组件的生命周期

页面生命周期的钩子只能在带有@Entry装饰的页面使用，组件生命周期钩子可以在任何页面使用

UIAbility的启动模式

Singleton启动模式

每一个UIAbility只存在唯一实例，是默认启动模式。任务列表中只会存在一个相同的UIAbility

standard和multion启动模式

每次启动UIAbility都会创建一个新的实例。
区别：multion再次启动会将前一个实例销毁，任务列表中始终只有一个UIAbility；
而standard再次启动不会销毁前一个实例，在任务列表中可能存在一个或多个相同的UIAbility

specified启动模式

每个UIAbility实例可以设置key标识，启动UIAbility时，需要指定key，存在key相同实例直接被拉起，不存在则创建新实例

specified启动模式步骤

当前UIAbility调用startAbility方法拉起目标UIAbility
  1. 获取上下文
context = getContext(this) as common.UIAbilityContext
  1. 指定要跳转的UIAbility的信息
let want: Want={
  deviceId:'', //为空指当前设备
  bundleName:'com.zj.demo', //在appScope中的app.json5中
  moduleName:'entry', //在当前项目的module.json5中
  abilityName:'DocumentAbility',
  parameters:{
    //getInstanceKey:自定方法，生成目标UIAbility的实例
    instanceKey:'idx_'+this.index++
  }
}
  1. 尝试拉起目标UIAbility实例
this.context.startAbility(want)
在AbilityStage的生命周期回调中为目标UIAbility实例生成key
import AbilityStage from '@ohos.app.ability.AbilityStage';
import Want from '@ohos.app.ability.Want';

export default class MyAbilityStage extends  AbilityStage{

  onAcceptWant(want:Want):string{
    if(want.abilityName==='DocumentAbility'){
      return `DocAbilityInstace_${want.parameters.instanceKey}`
    }
    return ''
  }
}
在module.json5配置文件中，通过srcEntry参数指定AbilityStage路径
{
  "module": {
    "name": "entry",
    "type": "entry",
    "srcEntry": './ets/myabilitystage/MyAbilityStage.ts'
  }
}
